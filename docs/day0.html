1

Antenna — get it outside

~1–2 h

The single biggest lever you have. Roughly 10–20 dB for $0–15 — worth more than every line of code in this project. Do it now, or accept a weak week.

  • Antenna mounted outdoors, as high and as clear of the house as you can manage. Indoors costs you more signal than any antenna upgrade can win back.
  • Horizontal dipole, cut for 104.3 MHz — each leg ≈ 27 in / 68 cm. Half-wave dipole: 143 / f(MHz) = 1.37 m total, split into two legs.
  • Broadside of the dipole aimed southeast, toward Boise (KAWO, ~650 km). A dipole is deaf off its ends — point those away from the transmitter.
  • Coax run is as short as practical; connectors dry and snug; nothing swinging in wind. A moving antenna modulates the signal and fakes pings.
  • Photograph the installation and note the aim. You will need to reproduce this exactly after any future change.
2

Verify the chain

~30 min

Prove signal reaches Python before you trust anything downstream.

  • Dongle opens, IQ flows, tuner reports sane values.
    python check_device.py
  • 104.3 MHz is still locally vacant — no local station has appeared.
    python fm_vacancy.py
    Stop if 104.3 is now occupied. Pick another vacant channel that still carries a distant station, and update reference_freq_hz.
  • Ten-minute capture: the noise floor is flat.
    python monitor.py 600
    Open monitor_power.png. Want: a flat line. A wandering floor means the baseline is chasing something — find it now, or every ping this week is junk.
  • Optional: measure the crystal error and set freq_correction_ppm.
    rtl_test -p
    Not critical — detection is wideband (±90 kHz), so drift doesn't bite here.

4

Prepare the machine

~20 min

On the Raspberry Pi — the intended host. The service already runs the low-memory profile (METEOR_PROFILE=pi, ~100 MB RSS).

  • Installed, service enabled, starts on boot.
    bash deploy/pi/install.sh sudo systemctl enable --now meteor-tracker
  • Power is clean — want 0x0.
    vcgencmd get_throttled
    Anything else = undervoltage. Fix the PSU before the run; weak power causes USB read errors.
  • Free disk ≥ 2 GB. df -h Snapshots are ~1–2 MB each as .npz; a busy week accumulates.

On Windows, if you're running there instead — the OS is actively trying to interrupt you.

  • System sleep and hibernation disabled.
  • USB selective suspend disabled. It will power-manage the dongle out from under you.
  • Windows Update paused for the week. A restart at 03:00 ends the run.
  • ≥ 1 GB free disk (PNG snapshots are far larger than the Pi's .npz).
5

Shakedown night — this is not data

1 night

The auto-reconnect and low-memory code has never faced real hardware overnight. Find out over one night, not over seven.

  • Start it and leave it alone.
    sudo systemctl start meteor-tracker journalctl -u meteor-tracker -f # watch the 5-min heartbeat
  • Confirm within 15 minutes that pings, CSV rows, and snapshots are all flowing — then walk away.

Morning go / no-go

  • Survived the night? Process still up, no gap in the heartbeat.
  • Reconnects: a few are fine and prove the recovery works. Dozens means a power or USB problem — fix it.
  • Memory flat? RSS steady near ~100 MB. Steadily climbing = a leak; stop and report it.
  • Snapshots dropped: a burst is fine (events are still logged). Constant dropping means the queue never drains.
  • Disk: extrapolate one night × 7. Will it fit?
  • sessions.csv exists and its exposure roughly matches the hours you actually ran.

All green → start the week. Anything red → fix it, then re-run the shakedown. A clean night of nothing beats a week of garbage.

6

Rules for the week ahead

Days 1–7
RuleWhy it exists
Run 24/7 Not just at night. Rate = counts ÷ exposure, and you need daytime coverage for the curve to mean anything. Daytime is also where the apex minimum and the radio-only daytime showers live.
Touch nothing Any change to antenna, gain, threshold, or frequency starts a new baseline — and the week over.
5 min each morning bash deploy/pi/report.sh and df -h. Log anything unusual: storms, power blips, an E-skip opening.
Plot the control On Day 9: python diurnal.py --class aircraft. Aircraft follow a human daily rhythm. If your meteor curve tracks the aircraft curve, the classifier is leaking and you're plotting SeaTac.
Distrust a perfect curve If diurnal.py ever stamps "EXPOSURE GUESSED" on the plot, the exposure log is missing and the curve may be an artifact of your own schedule. Do not submit it.
Success on Day 9 is a validated instrument, not a discovery. Perseids: Aug 12–13
